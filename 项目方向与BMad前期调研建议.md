# MyStartupProject1 项目方向与 BMad 前期调研建议

## 当前结论

- `康复方向` 保留为备选，不再作为当前主线
- 当前主线转为 `娱乐方向`
- 最优先验证的方向是 `AI 互动剧情 / 角色扮演产品`

## 当前判断依据

- 你更适合做 `线上获客 + 可全球化 + 低渠道依赖` 的产品
- 康复方向真实需求存在，但商业上更依赖渠道、机构合作或强信任背书
- 这和你当前 `不想跑合作`、`启动资金有限` 的约束不匹配

## 娱乐方向筛选标准

- `线上获客`
- `长使用周期`
- `低渠道依赖`
- `可全球化`
- `可用现有前端能力快速上线`

## 3 个更适合的娱乐方向

### 1. AI 互动剧情 / 角色扮演产品

- 一句话定位：面向喜欢角色陪伴和剧情沉浸的年轻用户，提供可持续聊天、剧情推进与关系成长的互动产品
- 核心页面 / 流程：
1. `角色广场`：选角色、看设定、进入首个剧情钩子
2. `剧情聊天页`：聊天推进主线、分支、好感与事件
3. `关系/剧情进度页`：查看章节、关系变化、解锁付费剧情包
- 判断：`优先级最高`

### 2. 单一长线游戏的 Companion 产品

- 一句话定位：围绕一个长线热门游戏，提供任务、配队、资料和进度管理的一站式伴随工具
- 核心页面 / 流程：
1. `今日任务首页`：每日提醒、活动倒计时、养成清单
2. `配队/资料库`：角色资料、阵容推荐、资源规划
3. `个人进度页`：收藏、养成记录、抽卡/收集追踪
- 判断：`更稳，但天花板取决于所选游戏`

### 3. 可分享的轻娱乐生成产品

- 一句话定位：用低门槛模板生成可分享的测验、梗图、角色设定或小剧情内容，靠传播拿到自然增长
- 核心页面 / 流程：
1. `模板选择页`：选择玩法模板和风格
2. `生成编辑页`：输入关键词并快速生成可玩内容
3. `分享落地页`：展示结果、二次传播、引导再生成
- 判断：`启动最快，但留存弱于前两者`

## 当前推荐

- 先验证 `AI 互动剧情 / 角色扮演产品`
- 原因：最符合 `长留存 + 全球化 + 内容驱动传播 + 订阅/内容包付费`
- 首版形态：做成 `app 形态的互动内容产品`，不是传统 `互动游戏`
- 第一版聚焦：`角色选择 -> 聊天推进剧情 -> 解锁下一段剧情`
- 第一版不要做：`复杂世界观编辑器`、`重数值系统`、`多人玩法`

## 为什么先做 App

- `更快上线`：更接近你擅长的内容型与前端产品形态
- `更容易付费`：`订阅 + 剧情包内购` 比传统游戏数值体系更轻
- `更容易冷启动`：先验证角色、剧情和付费点，不必先做完整游戏系统
- `更容易迭代`：如果后续留存成立，再逐步加重互动和游戏化

## 第一版题材收敛

### 最推荐的首发题材

- `现代都市恋爱陪伴`
- 判断：`最容易冷启动`

### 为什么是它

- `世界观成本最低`：不需要先解释复杂设定
- `情绪价值最直接`：用户一上来就能理解陪伴、暧昧、心动、守护
- `内容生产更轻`：单角色或双角色就能成立
- `付费点自然`：亲密剧情、专属支线、特殊结局都容易收费

## 3 个最适合的剧情切口

### 1. 现代都市恋爱陪伴

- 目标用户细分：`18-30 岁女性`，喜欢乙游、韩剧感、角色陪伴、睡前聊天
- 剧情切口：`高情绪价值 + 单男主/少角色 + 陪伴关系逐步升温`
- 首个付费点：`解锁高甜支线 / 深夜专属剧情 / 好感突破章节`
- 结论：`第一版优先做这个`

### 2. 都市异能守护者

- 目标用户细分：喜欢一点幻想感，但不想看重设定硬核世界观的用户
- 剧情切口：`现代都市 + 守护关系 + 轻悬念事件`
- 首个付费点：`角色隐藏身份线 / 危机救援章节 / 特殊结局`
- 结论：`可做第二题材扩展`

### 3. 悬疑调查搭档

- 目标用户细分：喜欢推理、危险关系、暧昧拉扯的用户
- 剧情切口：`你与角色一起查案或追真相`
- 首个付费点：`关键案件章节 / 真相结局 / 角色秘密线`
- 结论：`记忆点强，但写作与结构成本更高`

## 当前最小产品定义

- 目标用户：先聚焦 `18-30 岁女性情绪陪伴用户`
- 题材：先做 `现代都市恋爱陪伴`
- 关系模式：先做 `1 个主角色`
- 免费策略：`首段相遇 + 少量日常推进`
- 首个付费点：`第一个关系突破剧情包`

## 免费 AI API 判断

- `乙游`：通常指面向女性用户的恋爱互动叙事产品
- 你当前更适合做：`轻乙游化的聊天陪伴 app`
- 结论：`免费 AI API 可以做原型，不适合做正式产品的长期聊天底座`

### 为什么不适合直接依赖免费 API

- `OpenAI API`：当前没有稳定官方免费层
- `Claude API`：当前没有官方免费 API 层
- `Gemini API`：有免费层，但限流、配额和条款更适合开发者/小项目，不适合把长期商业产品完全压在免费层上
- `OpenRouter free`：可用于原型，但官方也明确说明免费模型有 `rate limits`、`availability`、`latency` 和模型随机性问题

### 对第一版的建议

- 不要把第一版设计成 `无限自由聊天`
- 改做 `强约束聊天`：预设人设、短上下文、有限轮次、章节驱动
- 更稳的方案：`预写剧情分支 + 少量 AI 润色/回复`
- 如果坚持零成本起步，AI 只用于 `小流量内测`，不要承诺长期无限聊
